# Report Template — SKILL_AUDIT_REPORT.md

This document defines the output format for audit reports generated by the Solana Skill Quality Gate. Reports can be generated in **Markdown** (human-readable) or **JSON** (machine-readable) format.

---

## Generating a Report

```bash
# Markdown report (default)
node scripts/skillqa.mjs report ./path-to-skill --out SKILL_AUDIT_REPORT.md

# JSON report
node scripts/skillqa.mjs report ./path-to-skill --format json --out SKILL_AUDIT_REPORT.json

# Both formats
node scripts/skillqa.mjs report ./path-to-skill --format all --out-dir ./reports/
```

## Report Customization

### Verbosity Levels

```bash
# Summary only (score + band + PR readiness)
node scripts/skillqa.mjs report ./path --verbosity summary

# Standard (default — includes gate scores and recommendations)
node scripts/skillqa.mjs report ./path --verbosity standard

# Detailed (includes all check-level findings and evidence)
node scripts/skillqa.mjs report ./path --verbosity detailed

# Full (includes scan metadata, file list, and timing)
node scripts/skillqa.mjs report ./path --verbosity full
```

### Filtering

```bash
# Report on specific gates only
node scripts/skillqa.mjs report ./path --gates 3,4

# Report only failures
node scripts/skillqa.mjs report ./path --failures-only

# Report with specific output path
node scripts/skillqa.mjs report ./path --out ./reports/my-skill-audit.md
```

## Integration with CI/CD

The JSON report format is designed for CI/CD pipeline integration:

```yaml
# Example: GitHub Actions step
- name: Skill Quality Gate
  run: |
    REPORT=$(node scripts/skillqa.mjs report ./skill --format json --out /dev/stdout)
    SCORE=$(echo "$REPORT" | jq '.score.total')
    PR_READY=$(echo "$REPORT" | jq '.pr_readiness.ready')
    
    echo "Score: $SCORE"
    echo "PR Ready: $PR_READY"
    
    if [ "$PR_READY" != "true" ]; then
      echo "::error::Skill did not pass quality gate (score: $SCORE)"
      exit 1
    fi
```

### Exit Codes

| Code | Meaning                                          |
|------|--------------------------------------------------|
| 0    | Audit passed — skill is PR-ready                 |
| 1    | Audit failed — skill is not PR-ready             |
| 2    | Critical safety finding — immediate attention     |
| 3    | Invalid input — skill path not found or invalid   |
